# ai-Dashboard UI/UX 개선 및 기능 통합 계획

## 📊 현황 분석

### ✅ 현재 구현된 기능
- **가계부 (Ledger)**: index.html - 거래 관리, 카테고리, 자산, 예산 추적
- **고정비 관리**: fixed.html - AI 구독료, 일반 고정비 관리
- **백엔드 API**: FastAPI 기반 REST API
- **텔레그램 봇**: SMS 파싱 및 자동 저장
- **다중 통화**: KRW/USD/JPY 지원, 환율 자동 동기화

### ⚠️ 부족한 기능
1. 두 대시보드 간 UI/UX 불일치 (index.html vs fixed.html)
2. 페이지 간 네비게이션 미흡
3. 통합 대시보드 부재
4. 데이터 연동 및 시각화 미흡
5. 반응형 디자인 개선 필요
6. 모바일 UX 미흡

#### 2.1 통합 대시보드 개발
- [ ] `/frontend/dashboard.html` 생성

    - 전체 자산

    - 좌하: 일별/주별 지출 추이 (선형/막대)
    - 우하: 고정비 개별 항목 목록

  - **빠른 액션**
    - 거래 추가 버튼
    - 구독 추가 버튼

#### 2.2 가계부 페이지 개선 (index.html)

- [ ] 상세 거래 편집 개선

#### 2.3 고정비 페이지 개선 (fixed.html → subscriptions.html)

## 🛠 기술 스택 (추가/개선)

### 프론트엔드
- **Bundle**: Vite (빌드 성능 개선)
- **CSS Framework**: Tailwind CSS (이미 사용 중)
- **UI Components**: Custom components + open-source (Headless UI 등)
- **Charts**: Chart.js (현재) → Recharts 검토
- **State Management**: LocalStorage → 필요 시 Redux/Zustand
- **Testing**: Vitest + Playwright

### 백엔드
- **Database**: FastAPI + JSON → SQLite/PostgreSQL 검토
- **Caching**: Redis (선택사항)
- **Task Queue**: Celery + Redis (선택사항)
- **Monitoring**: Sentry (에러 로깅)
